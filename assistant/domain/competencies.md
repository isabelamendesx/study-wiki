# Áreas de Engenharia de Software (Competências)

Slugs usados no campo `competencies` do frontmatter e como tags. Representam áreas de conhecimento que as páginas da wiki cobrem.

## Slugs

```
product-acumen, data-driven, technical-communication, problem-decomposition,
domain-knowledge, technical-breadth, technical-depth, persistence, code-quality,
efficiency, operation, security, infrastructure, performance, architecture,
ai-engineering
```

## Descrições

| Slug | O que significa |
|---|---|
| `product-acumen` | Traduzir necessidades reais em soluções técnicas e medir o sucesso delas. Entender regras de negócio, KPIs e balancear qualidade com velocidade. |
| `data-driven` | Usar dados para guiar decisões: coletar métricas, analisar padrões e monitorar features contra objetivos. |
| `technical-communication` | Expressar ideias técnicas com clareza para diferentes públicos via docs, diagramas, apresentações e discussões. |
| `problem-decomposition` | Quebrar problemas complexos em partes gerenciáveis e priorizadas. Aplicar 80/20 para focar no que gera mais impacto. |
| `domain-knowledge` | Conhecimento profundo dos produtos, plataformas e como seu domínio se interconecta com outras áreas e o negócio. |
| `technical-breadth` | Amplitude de conhecimento em múltiplas linguagens, ferramentas, arquiteturas e padrões. Aprendizado contínuo e adaptabilidade. |
| `technical-depth` | Domínio da stack principal: detalhes da linguagem, gerenciamento de memória, compilação e boas práticas ponta a ponta. |
| `persistence` | Modelagem e armazenamento de dados: escolha entre SQL/NoSQL, otimização de queries e índices, replicação e transações. |
| `code-quality` | Testes (unitários, integração, E2E, stress), código limpo, design patterns, análise estática e quality gates. |
| `efficiency` | Organização do trabalho e priorização: Scrum, Kanban, gestão de tempo, e saber quando tomar riscos responsáveis. |
| `operation` | Monitoramento, alertas, debugging, resposta a incidentes e post-mortems. Construir sistemas fáceis de operar. |
| `security` | Criptografia, desenvolvimento seguro (OWASP, AppSec), privacidade de dados (LGPD/GDPR) e prevenção de ameaças. |
| `infrastructure` | Serviços cloud (AWS), containers (Kubernetes), IaC (Terraform, Helm), admin Linux e debugging de infra. |
| `performance` | Profiling, teste de stress, monitoramento de performance e otimização de código/infra/processos para responsividade. |
| `architecture` | Design de sistemas escaláveis, seguros e distribuídos. Avaliação de tecnologias, documentação de decisões (ADRs) e requisitos não-funcionais. |
| `ai-engineering` | Uso de ferramentas de IA (copilots, agentes, LLMs) para melhorar fluxos de engenharia de software. Prompt engineering, MCP e AI safety. |

> `ai-engineering` é o tema transversal de estudos do usuário (code agents, AI copilots, etc.).
