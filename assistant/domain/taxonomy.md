# Taxonomia de Tags

Use apenas tags da taxonomia aprovada abaixo. A LLM pode propor adições (durante um lint), mas não deve inventar tags ad-hoc.

## Áreas de Engenharia de Software

`product-acumen`, `data-driven`, `technical-communication`, `problem-decomposition`,
`domain-knowledge`, `technical-breadth`, `technical-depth`, `persistence`,
`code-quality`, `efficiency`, `operation`, `security`, `infrastructure`,
`performance`, `architecture`

## IA Aplicada a Engenharia de Software

`ai-engineering`, `code-agents`, `ai-copilots`, `prompt-engineering`, `llms`,
`mcp`, `ai-safety`, `ai-evals`

## Fundamentos de Engenharia de Software

`design-patterns`, `testing`, `refactoring`, `concurrency`, `data-structures`,
`algorithms`, `distributed-systems`, `databases`, `devops`, `api-design`,
`observability`

## Stack / Linguagens

`golang`, `java`, `kotlin`, `spring-boot`, `kafka`, `redis`, `kubernetes`, `datadog`
